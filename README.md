# 🌱 지양하월시아 방주 프로젝트 — 디지털 보증서 시스템

제품(고유)번호를 입력하면 **럭셔리 디지털 보증서 페이지**가 자동으로 만들어지고,
고객에게 **알림톡(또는 문자)으로 보증서 링크가 자동 발송**되는 1인 운영용 웹사이트입니다.
손님은 공개 페이지에서 **고유번호만 입력하면 직접 보증서를 조회**할 수 있고, 모든 **발송 내역이 자동 기록**됩니다.

---

## 0. 화면 구성

- **/** (손님용 공개 조회) : 고유번호 입력 → 정품 보증서 확인
- **/certificate/(고유번호)** : 보증서 페이지 (등급 엠블럼·사진·육종정보·소유이력)
- **/admin** : 관리자 (로그인 후 사용)
  - **개체 등록** : 새 개체 정보 입력 → 구글 시트에 자동 저장
  - **보증서 발송** : 제품번호 + 고객번호 → 알림톡/문자 발송
  - **발송 내역** : 발송 기록 보기 / 수동 추가 / 삭제

---

## 1. 준비물 (모두 무료 시작 가능)

- [ ] GitHub 계정 — 코드 보관
- [ ] Google 계정 — 구글 시트 + 서비스 계정
- [ ] Railway 계정 — 웹사이트 구동
- [ ] 솔라피(Solapi) 계정 — 알림톡/문자 발송

> ⚠️ **알림톡을 어느 업체에서 만드셨는지 먼저 확인하세요.** 솔라피면 그대로 사용 가능, 다른 업체면 솔라피에 재연동하거나 `lib/messaging.js` 발송부만 교체합니다. (알림톡 값을 비워두면 처음엔 문자로 자동 발송)

---

## 2. 구글 시트 만들기 (데이터 저장소)

1. sheets.google.com → 새 스프레드시트 → 이름 `방주_개체관리`
2. 시트(탭) 이름을 **`Plants`** 로 변경
3. **1행(맨 윗줄)에 아래 16개 제목을 그대로** 입력 (순서 바뀌어도 OK, 글자만 정확히)

```
고유번호 | 품종명 | 영문명 | 등급 | 종 | 육종가 | 육종연도 | 모본 | 부본 | DNA마커 | 사진URL | 소유자 | 소유이력 | 발급일 | 관리자메시지 | 상태
```

> 💡 제목 줄만 만들고, 데이터는 **개체 등록 폼**에서 입력하면 자동으로 채워집니다.
> 첨부한 `Plants_시트_예시.csv`를 [파일 → 가져오기]로 올리면 제목+예시가 한 번에 들어갑니다.

**칸 설명 (예시값)**
- **고유번호**: 제품번호. 형식 예 `JY-PYG-JY001-26-RG0761` ( JY-계통-개체-연도-RG일련 ). 보증서를 찾는 열쇠, 중복 금지.
- **품종명 / 영문명**: 한글명(예: 러브레터) / 영문명(예: Love Letter).
- **등급**: 그린 / 레드 / 골드 / 로얄골드 중 하나. 보증서에 등급 엠블럼으로 표시됩니다. (보통 로얄골드만 발급)
- **종**: 종/계통. 예 `H.springboks hyb`.
- **육종가/육종연도/모본/부본/DNA마커**: 보증서에 표시. 비우면 그 줄은 숨김.
- **사진URL**: 개체 사진 인터넷 주소 (`.jpg`/`.png` 권장).
- **소유자**: 현재 소유자 (발송 시 자동 기록도 됨).
- **소유이력**: 소유자 이전 기록. **한 줄에 하나씩**, `날짜 | 내용` 형식. 예) `2026 | 최초 육종 (지양하월시아)`. 보증서 타임라인으로 표시.
- **발급일**: 발송 시 자동 기록.
- **관리자메시지**: 비우면 기본 환영 문구.
- **상태**: 예) `정품 인증`.

> 🗂 **Logs 탭은 자동 생성됩니다.** 보증서를 처음 발송하면 발송 내역을 저장할 `Logs` 탭이 시트에 자동으로 만들어집니다. 직접 만들 필요 없습니다.

4. 시트 주소에서 **시트 ID** 메모: `https://docs.google.com/spreadsheets/d/`**`이부분`**`/edit`

## 3. 구글 서비스 계정 발급 (한 번만)

1. console.cloud.google.com → 새 프로젝트 만들기 → 선택
2. **Google Sheets API** 검색 → 사용 설정(Enable)
3. **서비스 계정(Service Account)** 만들기 → 이름 아무거나 → 완료
4. 만든 서비스 계정 → **키(KEYS)** 탭 → 키 추가 → 새 키 → **JSON** → 파일 다운로드 (잘 보관)
5. JSON 안의 `client_email`(…iam.gserviceaccount.com) 복사
6. **구글 시트 → 공유** → 그 이메일을 **편집자**로 공유

---

## 4. 솔라피(Solapi) 설정

1. solapi.com 가입 → **발신번호 등록**(본인 번호 인증) → `SENDER_PHONE`
2. 콘솔 **개발/연동 → API Key** 에서 **API Key / Secret** 발급 → `SOLAPI_API_KEY`, `SOLAPI_API_SECRET`
3. (선택) 카카오 채널 연동 후 **템플릿 ID / 발신프로필(pfId)** → `ALIMTALK_TEMPLATE_ID`, `KAKAO_PFID`
   - 비우면 자동으로 문자(LMS) 발송. 알림톡 준비되면 채우세요.

---

## 5. GitHub 업로드 → Railway 배포

1. GitHub에 새 저장소 만들고 이 폴더 **안의 파일들**(server.js, package.json, lib, views 등)을 업로드. (`.env`는 올리지 않음)
2. railway.com → GitHub 로그인 → **New Project → Deploy from GitHub repo** → 저장소 선택
3. 배포되면 **Settings → Networking → Generate Domain** 으로 주소 생성
4. **Variables** 탭에 환경변수 입력:

| 변수 | 값 |
|---|---|
| `ADMIN_PASSWORD` | 관리자 비밀번호 |
| `BASE_URL` | 생성된 도메인 (끝에 / 없이) |
| `SHEET_ID` | 구글 시트 ID |
| `SHEET_NAME` | `Plants` |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | 서비스계정 JSON 전체 |
| `SOLAPI_API_KEY` / `SOLAPI_API_SECRET` | 솔라피 키 |
| `SENDER_PHONE` | 발신번호 |
| `KAKAO_PFID` / `ALIMTALK_TEMPLATE_ID` | (알림톡 쓸 때만) |
| `SESSION_SECRET` | 아무 긴 랜덤 문자열 |

> GitHub 저장소를 수정해서 다시 올리면(커밋하면) Railway가 **자동으로 다시 배포**합니다.

## 6. 사용하기

- **손님**: `내도메인/` 접속 → 고유번호 입력 → 보증서 확인. (또는 받은 링크/QR로 바로 접속)
- **관리자**: `내도메인/admin` 로그인 →
  - **개체 등록**: 품종·등급·종·육종·소유이력 입력 → 시트에 자동 추가 (다음 RG 일련번호 추천 표시)
  - **보증서 발송**: 고유번호 + 고객번호 → 알림톡/문자 발송 (소유자·발급일·발송내역 자동 기록)
  - **발송 내역**: 기록 보기 / 수동 추가 / 삭제

---

## 7. 자주 묻는 문제

- **개체를 못 찾음** → 시트 `고유번호`와 입력값이 정확히 같은지(공백/대소문자) 확인.
- **사진이 안 보임** → `사진URL`이 진짜 이미지 주소인지(`.jpg`/`.png`) 확인.
- **발송 실패** → 발신번호 등록, 솔라피 잔액(충전), API 키 확인. 화면에 솔라피 오류가 그대로 표시됨.
- **문자로만 감** → `KAKAO_PFID`/`ALIMTALK_TEMPLATE_ID`가 비면 정상(문자). 알림톡 원하면 채우기.
- **Logs 탭 관련 오류** → 서비스 계정이 시트에 **편집자**로 공유됐는지 확인(탭 자동 생성에 편집 권한 필요).

## 8. 폴더 구성

```
haworthia-ark/
├─ server.js              ← 메인 서버 (라우팅/로그인/발송/내역)
├─ package.json
├─ .env.example / .gitignore
├─ README.md / Plants_시트_예시.csv
├─ lib/
│  ├─ sheets.js           ← 구글 시트 읽기/쓰기 + Logs 자동생성/삭제
│  └─ messaging.js        ← 솔라피 알림톡/문자 발송
└─ views/
   ├─ lookup.ejs          ← 손님 공개 조회 페이지
   ├─ certificate.ejs     ← 보증서 (등급 엠블럼 포함)
   ├─ register.ejs        ← 개체 등록 폼
   ├─ admin.ejs           ← 보증서 발송
   ├─ logs.ejs            ← 발송 내역 (추가/삭제)
   ├─ login.ejs           ← 관리자 로그인
   └─ not_found.ejs       ← 번호 없음 안내
```

막히는 화면이나 오류 메시지를 알려주시면 그 단계부터 같이 풀어드리겠습니다.
