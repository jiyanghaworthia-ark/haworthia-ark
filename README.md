# 🌱 지양하월시아 방주 프로젝트 — 디지털 보증서 시스템

제품(고유)번호만 입력하면, 그 개체의 **럭셔리 디지털 보증서 페이지**가 자동으로 만들어지고,
고객에게 **알림톡(또는 문자)으로 보증서 링크가 자동 발송**되는 1인 운영용 웹사이트입니다.

> 코딩을 몰라도 따라 할 수 있도록 한 단계씩 적었습니다. 천천히 순서대로만 하시면 됩니다.

---

## 0. 전체 흐름 한눈에 보기

```
[관리자(명준님)]
   │  1) 관리자 페이지에서 제품번호 + 고객 전화번호 입력
   ▼
[이 웹사이트(Railway 서버)]
   │  2) 구글 시트에서 그 번호의 개체 정보를 찾음
   │  3) 보증서 링크(.../certificate/HW-2026-0001) 생성
   ▼
[솔라피] ──4) 알림톡/문자 발송──▶ [고객 휴대폰]
                                      │  링크 클릭
                                      ▼
                          [럭셔리 디지털 보증서 페이지]
```

데이터(개체 목록)는 **구글 시트**에 저장됩니다. 명준님이 시트에서 눈으로 보고 직접 수정할 수 있습니다.

## 1. 준비물 체크리스트

시작 전에 아래 4개 계정만 있으면 됩니다. (모두 무료로 시작 가능)

- [ ] **GitHub** 계정 — 코드 보관 (github.com)
- [ ] **Google(구글)** 계정 — 구글 시트 + 서비스 계정
- [ ] **Railway** 계정 — 웹사이트 실제 구동 (railway.com)
- [ ] **솔라피(Solapi)** 계정 — 알림톡/문자 발송 (solapi.com)

> ⚠️ **가장 먼저 확인할 것 — 알림톡을 어디서 만드셨나요?**
> "알림톡까지 완료"하셨다면, 그 알림톡 **템플릿/카카오 채널이 어느 발송업체에 등록**되어 있는지 꼭 확인하세요.
> - 솔라피에서 만드셨다면 → 이 코드 그대로 사용 가능합니다. 👍
> - 다른 업체(알리고, NHN, 다른 대행사 등)에서 만드셨다면 → 두 가지 중 하나입니다.
>   ① 그 업체에서 만든 채널/템플릿을 **솔라피에 다시 연동**하거나,
>   ② `lib/messaging.js`의 발송 부분만 그 업체 API로 교체 (제가 도와드릴 수 있습니다).
> 확실하지 않으면 알려주세요. 같이 확인해 드리겠습니다.

## 2. 1단계 — 코드를 GitHub에 올리기

1. github.com 로그인 → 오른쪽 위 `+` → **New repository**
2. 이름은 `haworthia-ark` 같은 걸로, **Private(비공개)** 선택 → **Create repository**
3. 방금 받은 `haworthia-ark` 폴더 전체를 올립니다. 가장 쉬운 방법:
   - 생성된 저장소 화면에서 **"uploading an existing file"** 링크 클릭
   - `haworthia-ark` 폴더 **안의 파일들**(server.js, package.json, lib 폴더, views 폴더 등)을 드래그해서 업로드
   - 맨 아래 **Commit changes** 클릭
4. `.env` 파일은 **절대 올리지 마세요.** (비밀 정보라서 `.gitignore`가 자동으로 막아줍니다)

> 💡 더 편한 방법을 원하시면 "GitHub Desktop" 프로그램을 쓰면 폴더째로 올리기 쉽습니다.

## 3. 2단계 — 구글 시트 만들기 (데이터 저장소)

1. sheets.google.com → **빈 스프레드시트** 새로 만들기
2. 이름을 `방주_개체관리` 등으로 변경
3. 왼쪽 아래 시트(탭) 이름을 **`Plants`** 로 변경 (대소문자 정확히)
4. **1행(맨 윗줄)에 아래 제목들을 그대로** 입력하세요. (순서는 바뀌어도 됩니다. 글자는 정확히)

| 고유번호 | 품종명 | 육종가 | 육종연도 | 모본 | 부본 | DNA마커 | 사진URL | 소유자 | 소유이력 | 발급일 | 관리자메시지 | 상태 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|

> 💡 시트를 직접 안 채워도 됩니다. 아래 **개체 등록 폼**에서 입력하면 이 줄들이 자동으로 추가됩니다.
> (열 제목만 정확히 만들어 두면 됩니다. 첨부한 `Plants_시트_예시.csv`를 가져오기 하면 가장 빠릅니다.)

5. 칸 설명 (예시값):

- **고유번호**: `HW-2026-0001` (HW-연도-일련번호). 보증서를 찾는 열쇠. 중복 금지.
- **품종명/육종가/육종연도/모본/부본/DNA마커**: 보증서에 그대로 표시. 비우면 그 줄은 숨겨짐.
- **사진URL**: 개체 사진의 인터넷 주소 (`.jpg`/`.png` 권장).
- **소유자**: 현재 소유자. (발송 시 자동 기록도 됨)
- **소유이력**: 소유자 이전 기록. **한 줄에 하나씩**, `날짜 | 내용` 형식. 예) `2024 | 최초 육종 (지양하월시아)` / `2026-06 | 홍길동 분양`. 보증서의 타임라인으로 표시됩니다.
- **발급일**: 발송 시 자동 기록.
- **관리자메시지**: 비우면 기본 환영 문구.
- **상태**: 예) `정품 인증`.

**각 칸 설명**
- **고유번호**: 보증서를 찾는 열쇠. 권장 형식 `HW-2026-0001` (HW-연도-일련번호). 중복 금지.
- **품종명 / 육종가 / 육종연도 / 모본 / 부본 / DNA마커**: 보증서에 그대로 표시됩니다. 비워두면 그 줄은 표시되지 않습니다.
- **사진URL**: 개체 사진의 **인터넷 주소**. (만드는 법은 아래 팁 참고)
- **소유자 / 발급일**: 발송할 때 시스템이 **자동으로 채워줍니다.** (직접 안 적어도 됨)
- **관리자메시지**: 비워두면 기본 환영 문구가 자동으로 들어갑니다.
- **상태**: 예) `정품 인증`. 비우면 기본값 `정품 인증`.

> 📷 **사진URL 만드는 가장 쉬운 방법**: 구글 드라이브 대신, 사진을 **imgur.com** 같은 곳에 올리거나,
> 쇼핑몰/블로그에 올린 이미지의 주소를 복사해 넣으세요. 주소가 `.jpg` / `.png` 로 끝나면 가장 안전합니다.
> (구글 드라이브 링크는 보증서에서 사진이 안 뜨는 경우가 많아 권장하지 않습니다.)

6. 시트 주소창에서 **시트 ID**를 메모해 두세요.
   `https://docs.google.com/spreadsheets/d/`**`이부분이_시트ID`**`/edit`

## 4. 3단계 — 구글 서비스 계정 발급 (시트를 코드가 읽게 허용)

이 단계가 조금 길지만, **딱 한 번만** 하면 됩니다.

1. **console.cloud.google.com** 접속 (구글 계정으로 로그인)
2. 상단에서 **새 프로젝트** 만들기 → 이름 `haworthia-ark` → 만들기 → 그 프로젝트 선택
3. 왼쪽 메뉴 검색창에 **"Google Sheets API"** 검색 → 들어가서 **사용 설정(Enable)** 클릭
4. 다시 검색창에 **"서비스 계정(Service Account)"** → **서비스 계정 만들기**
   - 이름: `ark-bot` 등 아무거나 → 만들고 계속 → 역할은 건너뛰어도 됨 → 완료
5. 만들어진 서비스 계정을 클릭 → 상단 **키(KEYS)** 탭 → **키 추가 → 새 키 만들기 → JSON** 선택
   → `.json` 파일이 컴퓨터에 다운로드됩니다. **이 파일이 비밀번호입니다. 잘 보관하세요.**
6. 그 JSON 파일을 메모장으로 열면 `"client_email": "ark-bot@....iam.gserviceaccount.com"` 같은 **이메일 주소**가 있습니다. 복사하세요.
7. **2단계에서 만든 구글 시트로 돌아가** → 오른쪽 위 **공유** 버튼 → 방금 복사한 그 이메일을 붙여넣고
   **편집자(Editor)** 권한으로 공유 → 완료.

> 이제 코드가 그 시트를 읽고/쓸 수 있게 됩니다. JSON 파일 내용은 5단계에서 Railway에 넣습니다.

## 5. 4단계 — 솔라피(Solapi) 설정 (발송)

1. **solapi.com** 가입 → 로그인
2. **발신번호 등록**: 메뉴에서 "발신번호" → 본인 휴대폰/사업자 번호를 등록(인증)합니다.
   → 등록한 번호를 `SENDER_PHONE` 으로 쓸 거예요. (예: `01012345678`)
3. **API 키 발급**: 콘솔의 **개발/연동 → API Key** 메뉴에서 **API Key**와 **API Secret**을 발급/복사.
   → `SOLAPI_API_KEY`, `SOLAPI_API_SECRET` 으로 씁니다.
4. **(선택) 알림톡 연동**: 카카오 채널을 솔라피에 연동하고, 승인된 **템플릿 ID**와 **발신프로필(pfId)**을 확인합니다.
   → `KAKAO_PFID`, `ALIMTALK_TEMPLATE_ID` 로 씁니다.
   - 이 두 값을 **비워두면 처음엔 자동으로 문자(LMS)로 발송**됩니다. 그래서 알림톡 준비 전에도 바로 운영을 시작할 수 있어요.
   - 알림톡 템플릿에 변수가 있다면(예: `#{고객명}`, `#{링크}`) 코드 기본값과 같은지 확인하세요. 다르면 `ALIMTALK_VAR_NAME`, `ALIMTALK_VAR_LINK` 로 맞춰주면 됩니다.

> 💰 솔라피는 충전식(선불)입니다. 문자 1건/알림톡 1건당 소액이 차감됩니다. 소량 충전으로 테스트부터 하세요.

## 6. 5단계 — Railway에 배포 (웹사이트 켜기)

1. **railway.com** → **GitHub로 로그인**
2. **New Project → Deploy from GitHub repo** → 아까 만든 `haworthia-ark` 저장소 선택
3. Railway가 자동으로 Node.js 앱임을 인식하고 설치/실행합니다. (`npm install` → `npm start` 자동)
4. **환경변수 입력**: 프로젝트 → **Variables** 탭 → 아래 값들을 하나씩 추가합니다.

| 변수 이름 | 넣을 값 |
|---|---|
| `ADMIN_PASSWORD` | 원하는 관리자 비밀번호 |
| `SHEET_ID` | 2단계에서 메모한 시트 ID |
| `SHEET_NAME` | `Plants` |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | 3단계 JSON 파일 **내용 전체**를 그대로 복사해 붙여넣기 |
| `SOLAPI_API_KEY` | 솔라피 API Key |
| `SOLAPI_API_SECRET` | 솔라피 API Secret |
| `SENDER_PHONE` | 솔라피에 등록한 발신번호 (예: 01012345678) |
| `KAKAO_PFID` | (알림톡 쓸 때만) 발신프로필 ID |
| `ALIMTALK_TEMPLATE_ID` | (알림톡 쓸 때만) 템플릿 ID |
| `SESSION_SECRET` | 아무 긴 랜덤 문자열 |

> `GOOGLE_SERVICE_ACCOUNT_JSON`은 `{` 로 시작해서 `}` 로 끝나는 JSON 전체를 **줄바꿈 포함 그대로** 붙여넣으면 됩니다.

5. **도메인 만들기**: 프로젝트 → **Settings → Networking → Generate Domain** 클릭
   → `https://haworthia-ark-production.up.railway.app` 같은 주소가 생깁니다.
6. 그 주소를 복사해서 **Variables에 `BASE_URL` 추가** (끝에 `/` 없이). 예: `https://haworthia-ark-production.up.railway.app`
   → 저장하면 자동으로 다시 배포됩니다.

## 7. 6단계 — 사용하기

1. 브라우저에서 `https://(내도메인)/admin` 접속 → `ADMIN_PASSWORD` 로 로그인
2. **[개체 등록]** 탭(`/admin/new`): 품종명·육종가·모본·부본·DNA마커·사진URL·소유자·소유 이력 등을 입력 → **개체 등록하기**
   → 구글 시트에 새 줄이 자동 추가됩니다. 고유번호는 **추천번호** 버튼으로 자동 생성됩니다.
3. **[보증서 발송]** 탭(`/admin`): **제품(고유)번호** + **고객 휴대폰 번호** 입력 → **보증서 발송하기**
4. 고객에게 알림톡(또는 문자)으로 보증서 링크가 도착 → 누르면 럭셔리 보증서 페이지가 열립니다.
5. 발송과 동시에 시트의 **소유자/발급일**이 자동으로 기록됩니다.

보증서 페이지를 직접 미리 보고 싶으면:
`https://(내도메인)/certificate/HW-2026-0001` 처럼 번호를 붙여 접속하면 됩니다.

## 8. 1인 기업 자동화 팁

- **QR코드/NFC**: 보증서 주소(`.../certificate/HW-2026-0001`)로 QR코드를 만들어 화분 택에 붙이면, 손님이 폰으로 찍어 바로 보증서를 봅니다. (네이버 QR, qrcode-monkey 등 무료 생성기 사용)
- **카톡 미리보기**: 링크를 카톡으로 보내면 사진+품종명이 카드 형태로 예쁘게 뜨도록 이미 설정해 두었습니다(Open Graph).
- **시트가 곧 관리대장**: 판매/소유 이력이 구글 시트에 자동 누적되므로 별도 장부가 필요 없습니다.
- **다음 확장(원하시면 추가 제작)**: 소유권 이전 버튼, 회원 관리, 통계 대시보드, 폐사 A/S 접수 폼, 거래제한 대상자 자동 차단 등 협의서 조항들을 단계적으로 붙일 수 있습니다.

## 9. 자주 묻는 문제 (Troubleshooting)

- **"개체를 시트에서 찾지 못했습니다"** → 시트의 `고유번호` 칸 값과 입력한 번호가 정확히 같은지 확인(공백/대소문자).
- **사진이 안 보임** → `사진URL`이 진짜 이미지 주소인지 확인. `.jpg`/`.png`로 끝나는 주소 권장.
- **발송 실패(솔라피 ...)** → 발신번호 등록 여부, 잔액(충전), API 키 정확성 확인. 화면에 솔라피 오류 메시지가 그대로 표시됩니다.
- **알림톡이 아니라 문자로 감** → `KAKAO_PFID` / `ALIMTALK_TEMPLATE_ID`가 비어있으면 정상 동작(문자 발송)입니다. 알림톡을 원하면 두 값을 채우세요.
- **시트 권한 오류** → 서비스 계정 이메일을 시트에 **편집자**로 공유했는지 확인(4단계 7번).

## 10. 폴더 구성 (참고)

```
haworthia-ark/
├─ server.js              ← 메인 서버 (경로/로그인/발송 처리)
├─ package.json           ← 필요한 부품 목록
├─ .env.example           ← 환경변수 예시 (복사해서 .env로 사용)
├─ .gitignore             ← 비밀파일이 깃에 올라가지 않게 막음
├─ lib/
│  ├─ sheets.js           ← 구글 시트 읽기/쓰기
│  └─ messaging.js        ← 솔라피 알림톡/문자 발송
└─ views/
   ├─ certificate.ejs     ← 고객용 럭셔리 보증서 페이지
```
